__block

  1. block 对外部变量的引用，block 默认是将其复制到其数据结构中来实现访问的
    如果这个对象是一个引用类型(指针).则 block 将其引用计数加1.
  2. 对于用 __block 修饰的外部变量,block 是复制其引用地址来实现访问的.

Block 中强引用和循环引用问题

ARC 下 Block 使用 copy 还是 strong

分析:
使用 copy 是因为在 MRC 的时候需要将栈区的 block 拷贝到堆区使用.防止block 出了函数作用域就被释放.
而现在 ARC 中,只要block 使用到外部变量,就默认是在堆区.不存在之前的问题.

答案. 用 copy 是没问题的.用 strong 也是没问题的.都是对 block 进行一个强引用而已.

Block 的循环引用

和控制器互相持有导致

分析: 有循环引用.首先 block 被控制器强引用. block 内部将 self 强引用.
导致控制器的引用计数无法为0,无法执行 dealloc方法.无法将 block 释放.
所以解决办法就是将其中的一个强引用打破.
